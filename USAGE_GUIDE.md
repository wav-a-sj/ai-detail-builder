## 시작하기

### 1. Gemini API 키 설정

웨이브A를 사용하려면 먼저 Gemini API 키가 필요합니다.

1. **API 키 발급**
   - https://makersuite.google.com/app/apikey 방문
   - Google 계정으로 로그인
   - "Create API Key" 버튼 클릭
   - 발급된 API 키 복사

2. **환경 변수 설정**
   ```bash
   # 프로젝트 루트에 .env 파일 생성
   VITE_GEMINI_API_KEY=여기에_발급받은_API_키_붙여넣기
   ```

3. **확인**
   - 애플리케이션을 실행했을 때 API 키 에러가 없으면 성공!

## 썸네일 생성 가이드

### 단계별 가이드

#### Step 1: 썸네일 제작 페이지 이동
- 상단 네비게이션에서 **"썸네일 제작"** 클릭

#### Step 2: 상품 정보 입력
1. **상품명** (필수)
   - 예: "프리미엄 무선 이어폰"
   - 짧고 명확하게 작성

2. **상품 특징** (필수)
   - 예: "노이즈 캔슬링, 20시간 재생, IPX7 방수, 프리미엄 사운드"
   - 주요 특징을 쉼표로 구분하여 입력

3. **이미지 스타일** 선택
   - **모던 (Modern)**: 세련되고 현대적인 느낌
   - **미니멀 (Minimalist)**: 깔끔하고 단순한 디자인
   - **생동감 (Vibrant)**: 화려하고 역동적인 느낌
   - **우아함 (Elegant)**: 고급스럽고 우아한 분위기

4. **원본 이미지** (선택)
   - 제품의 실제 이미지가 있다면 업로드
   - 최대 5MB까지 지원

#### Step 3: 썸네일 생성
- **"썸네일 생성"** 버튼 클릭
- AI가 약 5-10초 내에 썸네일 생성

#### Step 4: 결과 확인 및 다운로드
1. 생성된 썸네일 미리보기
2. 프롬프트 확인 (상세보기 클릭)
3. **"다운로드"** 버튼으로 이미지 저장
4. **"새로 만들기"** 버튼으로 다시 시작

### 💡 썸네일 생성 팁

- **상품명**: 브랜드명보다는 제품의 핵심 가치를 강조
- **특징**: 3-5개의 핵심 특징만 입력 (너무 많으면 산만함)
- **스타일**: 타겟 고객의 취향에 맞는 스타일 선택
  - 20-30대 → Modern, Vibrant
  - 40대 이상 → Elegant, Minimalist
  - B2B 제품 → Minimalist, Modern

## 상세페이지 제작 가이드

### 3단계 워크플로우

```
1. 기획 → 2. 생성 → 3. 결과
```

### Phase 1: 기획

#### Step 1: 상세페이지 제작 페이지 이동
- 상단 네비게이션에서 **"상세페이지 제작"** 클릭

#### Step 2: 상품 정보 입력

1. **기본 정보** (모두 필수)
   ```
   상품명: 프리미엄 무선 이어폰
   카테고리: 전자제품 > 오디오
   가격: 89000
   프로모션: 한정 특가 30% 할인 (선택)
   ```

2. **상품 특징** (필수)
   ```
   - 최신 ANC(능동형 노이즈 캔슬링) 기술
   - 20시간 장시간 재생
   - IPX7 방수 등급
   - 프리미엄 사운드 품질
   - 터치 컨트롤
   - 무선 충전 지원
   ```

3. **타겟 고객** (최소 1개 선택)
   - 연령대: 20대, 30대, 40대
   - 성별: 남성, 여성
   - 직업/상황: 직장인, 학생, 부모
   
   예: `20대`, `30대`, `직장인` 선택

4. **페이지 길이**
   - **자동 (Auto)**: AI가 자동 결정 (7장)
   - **짧게 (Short)**: 5장 - 간단한 제품
   - **표준 (Standard)**: 7장 - 일반적인 제품
   - **길게 (Long)**: 9장 - 복잡한 제품

#### Step 3: 기획 생성
- **"상세페이지 기획 생성"** 버튼 클릭
- AI가 10-15초 내에 기획안 생성

### Phase 2: 생성

#### Step 1: 기획안 확인
- 각 섹션의 제목, 핵심 메시지, 이미지 프롬프트 확인
- 필요시 **연필 아이콘**을 클릭하여 수정 가능

#### Step 2: 섹션 수정 (선택)
1. 수정하려는 섹션의 **연필 아이콘** 클릭
2. 제목, 핵심 메시지, 이미지 프롬프트 수정
3. **체크 아이콘**으로 저장 또는 **X 아이콘**으로 취소

#### Step 3: 이미지 생성
1. **"전체 이미지 생성 시작"** 버튼 클릭
2. 진행률 표시 확인
3. 섹션별로 순차적으로 생성 (각 1-2초)
4. 완료 후 **"생성 결과 보기"** 버튼 클릭

### Phase 3: 결과

#### 연속 뷰어
- 좌측: 모든 이미지를 세로로 이어서 확인
- 실제 상세페이지처럼 스크롤하며 확인 가능

#### 섹션별 상세
- 우측: 각 섹션의 상세 정보
- 개별 이미지 다운로드 가능
- 프롬프트 확인 가능

#### 다운로드
1. **전체 다운로드**: 모든 이미지를 순차적으로 다운로드
2. **개별 다운로드**: 각 섹션의 다운로드 버튼 클릭

#### 새로 만들기
- **"새로 만들기"** 버튼으로 처음부터 다시 시작

### 💡 상세페이지 제작 팁

#### 효과적인 특징 작성
```
❌ 좋지 않은 예:
"좋은 이어폰입니다"

✅ 좋은 예:
"최신 ANC 기술로 지하철 소음도 차단
20시간 연속 재생으로 출퇴근 걱정 없음
IPX7 방수로 운동 중에도 안심"
```

#### 타겟 설정 전략
- **좁게 설정**: 더 집중된 메시지 (예: 20대 남성 직장인)
- **넓게 설정**: 다양한 고객층 (예: 20-40대 직장인)

#### 페이지 길이 선택
- **5장 (Short)**: 단순한 제품, 빠른 결정
- **7장 (Standard)**: 일반적인 제품, 균형잡힌 설명
- **9장 (Long)**: 고가 제품, 상세한 설명 필요

#### 섹션 순서 이해
1. **Hook**: 관심 끌기
2. **Solution**: 문제 해결 제시
3. **핵심 가치**: 주요 장점
4. **기능 설명**: 상세 기능
5. **신뢰 구축**: 사회적 증거
6. **비교/차별점**: 경쟁사 대비 장점
7. **CTA**: 구매 유도

## 문제 해결

### Q1. "Gemini API 키가 설정되지 않았습니다" 에러

**해결방법:**
1. `.env` 파일이 프로젝트 루트에 있는지 확인
2. `VITE_GEMINI_API_KEY=실제키` 형식이 맞는지 확인
3. API 키 앞뒤에 공백이나 따옴표가 없는지 확인
4. 페이지 새로고침

### Q2. 이미지가 생성되지 않아요

**현재 상태:**
- 현재 버전은 Mock 이미지를 생성합니다
- 실제 이미지 생성은 추가 API 연동이 필요합니다

**해결방법:**
- Gemini Vision API나 이미지 생성 API를 추가로 연동

### Q3. 기획안이 이상하게 생성돼요

**해결방법:**
1. 상품 특징을 더 구체적으로 작성
2. 타겟 고객을 명확히 설정
3. 다시 생성 버튼 클릭
4. 생성된 기획안을 직접 수정

### Q4. 다운로드가 안 돼요

**해결방법:**
1. 브라우저 팝업 차단 확인
2. 다운로드 권한 허용
3. 개별 다운로드 시도

### Q5. 로딩이 너무 오래 걸려요

**정상 처리 시간:**
- 썸네일: 5-10초
- 기획안: 10-15초
- 이미지 생성 (7장): 10-15초

**30초 이상 걸린다면:**
1. 네트워크 연결 확인
2. API 키 확인
3. 페이지 새로고침 후 재시도
